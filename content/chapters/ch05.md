# 第五章：省钱篇 - 别让AI把你榨干

> 我有一天烧了80块，全在调试浏览器。这章教你怎么控制成本。

---

## 我的教训

2026年2月24日，我一天烧了约80元。

原因：反复调试浏览器自动化，每次失败都要重新跑，token 哗哗地流。

那天之后我立了一条规矩：**大操作前先估算 token，超预算先告知再做。**

---

## Token 是什么

Token 是 AI 处理文字的计量单位，大概：
- 1个汉字 ≈ 1-2个 token
- 1个英文单词 ≈ 1个 token
- 1000个 token ≈ 0.5-2元（取决于模型）

**越贵的模型，token 越贵，但能力越强。**

---

## 模型分级策略

不是所有任务都需要最贵的模型：

| 场景 | 推荐模型 | 原因 |
|------|---------|------|
| 日常聊天、简单问答 | QwenTurbo | 最便宜，够用 |
| 写文案、分析数据 | QwenPlus / claude-sonnet | 中等，性价比高 |
| 复杂策略、重要决策 | claude-sonnet | 贵但值 |
| 定时任务（cron） | QwenTurbo | 批量跑，省钱 |

切换模型命令：
```
/model QwenTurbo
```

---

## 渐进式披露：最有效的省钱技巧

**什么是渐进式披露？**

不要一次性把所有文件都塞给 AI，按需加载。

**错误做法：**
每次对话都把 MEMORY.md + 所有日记 + 所有配置文件全部加载进来。

**正确做法：**
- 先加载核心文件（SOUL.md、USER.md、MEMORY_HANDOVER.md）
- 需要具体信息时再读对应文件
- 不相关的文件不加载

**效果：** token 消耗减少 40-60%。

## 定时任务省钱

cron 任务每天跑多次，用便宜模型：

在 cron 配置里指定模型：
```json
{
  "model": "dashscope/qwen-turbo"
}
```
