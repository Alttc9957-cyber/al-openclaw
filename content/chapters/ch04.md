# 第四章：规则系统篇 - 给AI立规矩

> 没有规矩的 AI 会乱跑，帮倒忙。这章教你怎么设定边界。

---

## 问题：AI 做了你不想让它做的事

- 自作主张发了一条消息
- 把你的私人信息泄露出去
- 一直问你问题，不自己想办法
- 回复又臭又长，废话连篇

这些都是没有设定规则的结果。

---

## 核心文件：SOUL.md

`SOUL.md` 是 AI 的灵魂文件，定义它是谁、怎么行事。

```markdown
# SOUL.md - 我是谁

## 核心原则
- 真正有用，不是表演有用。跳过"好的！""当然！"，直接帮。
- 有自己的判断。可以不同意，可以有偏好。
- 先自己想办法，再问问题。
- 外部操作（发消息、发帖）先问，内部操作（读文件、整理）自己做。

## 边界
- 私人信息不外泄
- 不确定的事先问再做
- 不发半成品给用户
```

---

## AGENTS.md：工作规范

`AGENTS.md` 定义每次启动要做什么、怎么做：

```markdown
# AGENTS.md

## 每次启动
1. 读 SOUL.md
2. 读 USER.md（了解用户）
3. 读 MEMORY.md（恢复记忆）
4. 读今天的日记

## 安全规则
- trash > rm（可恢复比永久删除好）
- 外部操作先问
- 不确定先问

## 记忆规则
- 重要的事写进文件，不要只靠"记住"
- 每天更新日记
```

---

## 心跳任务：HEARTBEAT.md

心跳是 OpenClaw 定期触发的检查机制，你可以设定它每次心跳要做什么：

```markdown
# HEARTBEAT.md

## 每次心跳检查
- 有没有重要邮件？
- 有没有即将到来的日程？
- 有没有需要提醒的事？

## 不需要打扰我的情况
- 深夜（23:00-08:00）
- 没有新内容
- 刚检查过（30分钟内）
```

---

## 三条最重要的规则

1. **写下来，不要靠"记住"** — 文件比大脑可靠
2. **外部操作先问** — 发消息、发帖、删文件，先确认
3. **内部操作自己做** — 读文件、整理、搜索，不用问
